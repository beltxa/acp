# ACP Economic Model

## Core Insight

Protocols themselves are rarely monetizable.

Value is captured through:

- infrastructure
- network services
- enterprise capabilities

ACP is designed from the start with this layered model.

---

## Layered Ecosystem

| Layer | Description | Monetization |
|------|-------------|-------------|
| Protocol | Open ACP specification | None |
| Infrastructure | Relay and discovery services | Usage pricing |
| Enterprise | Governance, compliance, reliability | Premium services |

---

## Avoiding the AMQP Problem

AMQP became successful but the protocol creators did not capture the economic value.

ACP attempts to avoid this outcome by:

1. Keeping the protocol open
2. Building monetizable infrastructure layers
3. Providing enterprise-grade governance services

---

## Potential Revenue Streams

### Managed Relay Networks
Organizations pay for:

- high availability
- routing optimization
- operational monitoring

### Discovery Infrastructure
Hosted discovery and identity registries.

### Enterprise Governance Layer
Premium features such as:

- policy enforcement
- compliance logging
- deterministic delivery guarantees

---

## Adoption Strategy

1. Release protocol and SDKs openly
2. Encourage developer experimentation
3. Grow relay and discovery infrastructure
4. Introduce enterprise capabilities later

---

## Strategic Goal

Make ACP the default communication layer for autonomous agents while monetizing the infrastructure and enterprise ecosystem around it.
